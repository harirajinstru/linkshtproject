# Agent Instructions — Commits & PRs

Commit messages
- Use conventional, concise messages: `type(scope): short description`.
- Examples: `fix(db): correct index usage`, `feat(ui): add dark mode toggle`.

PR content
- Describe the change, list files modified, include testing steps, and call out any risk areas.
- Link to related issues where applicable and request specific reviewers.

Review & approval
- Small docs/code fixes: one reviewer.
- Risky changes (security, infra, migrations): require at least two reviewers and a human approval note.
